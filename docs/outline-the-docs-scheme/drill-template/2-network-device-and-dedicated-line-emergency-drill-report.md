# 网络设备与专线应急演练报告

## 演练简介

### 一、演练时间

20XX年X月X日

### 二、演练目的

为防止网络设备意外故障导致业务中断带来的影响

### 三、演练过程

1、模拟联通主线出现故障，观察是否出现生产故障，是否有告警以及主防火墙是否切至备墙，是否切换至电信备线。

2、模拟主防火墙故障，下行（上行）故障，观察是否出现生产故障，是否有告警以及主防火墙是否切至备墙。以及防火墙单心跳线故障，（当前心跳口只有一个，需要增加一个心跳口不能设置同一网段），是否有告警，是否出现生产故障，备用心跳口是否启用。

3、模拟主核心故障，下行（上行）故障，观察是否出现生产故障，观察路由路径是否发生变化（可能）。以及模拟核心堆叠线发生故障，观察是否出现生产故障，主备核心交换机是否切换主从。

4、模拟主汇聚故障，下行（上行）故障，观察是否出现生产故障，观察路由路径是否发生变化（可能）。以及模拟汇聚堆叠线发生故障（单条），观察是否出现生产故障，主备核心交换机是否切换主从。

5、主汇聚恢复，观察恢复过程以及恢复后是否出现生产故障，主备汇聚交换机是否切换主从。以及汇聚堆叠口恢复，观察恢复过程是否出现生产故障，主备汇聚交换机是否切换主从。

6、主核心恢复，观察恢复过程以及恢复后是否出现生产故障，主备核心交换机是否切换主从。以及主核心堆叠口恢复，观察恢复过程是否出现生产故障，主备核心交换机是否切换主从。

7、主防火墙恢复，观察恢复过程是否出现生产故障，是否有告警以及防火墙主备是否切换。防火墙心跳口恢复，观察恢复过程是否出现生产故障，心跳接口是否切换。

8、运营商恢复，观察恢复过程是否出现生产故障，是否有告警以及防火墙主备是否切换。

### 四、演练结果

经网络稳定性测试后，确定XX与YY的互联链路或XX端的防火墙-核心-汇聚这些重要设备出现单台设备故障时，对生产无影响。
