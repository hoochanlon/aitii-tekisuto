## 邮件点评

### 做得好的地方

| 方面 | 说明 |
|------|------|
| **主题与性质一目了然** | 标题用「【专线测试报备】+ 备线参数测试」点明是测试类报备而非割接/故障，首段写明「不涉及任何配置及设备变更」，收件人与甲方可快速判断风险等级和关注点。 |
| **诉求与动作明确** | 开篇即说明「烦请协助向甲方进行报备，内容如下」，收件人清楚自己的角色是「转述/提交给甲方」，正文可直接摘录或转发。 |
| **四块结构便于摘录** | 测试时间、影响范围、报备内容（屏蔽告警）、风险保障措施分块加粗，与割接报备格式一致，便于值班或甲方按项查阅、留档。 |
| **影响与时长可预期** | 「预计最长X小时」「计划X小时内完成」给出时间上界，便于甲方安排业务窗口和评估影响；「导致XX专线传输链路中断」写清影响范围。 |
| **保障措施可落地** | 运营商经理全程陪同、异常时XX分钟内恢复，以及施工后IT现场验证、确保次日业务正常，既有过程保障又有结果验证，便于甲方接受报备。 |

### 可补充（视单位要求选用）

- **落款与日期**：报备人、报备部门、日期，便于与测试记录、甲方报备材料对应留痕。
- **如需抄送**：若需同步甲方对接人、运维值班等，可注明抄送对象，确保测试窗口内各方信息一致。
- **测试后闭环**：若制度要求「测试结束后再发恢复/结果确认」，可约定「测试完成后将另行邮件/群内通报验证结果」，形成完整闭环。
