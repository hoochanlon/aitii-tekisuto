## 邮件点评

### 做得好的地方

| 方面 | 说明 |
|------|------|
| **主题与诉求开门见山** | 标题用「【专线割接报备】+ 运营商」点明类型与对象，首段即交代「接运营商通知」「线路将中断」和「烦请协助操作屏蔽告警」，收件人立刻知道要做什么、为何要做。 |
| **动作与渠道分工清楚** | 写清「报备信息将同步发送至工作群并通过邮件正式提交」，口头/群内同步与邮件留档各司其职，避免重复通知或遗漏。 |
| **四要素分条易扫读** | 中断时间、影响范围、报备内容（屏蔽告警及原因）、风险保障措施分条列出，每项加粗小标题，便于值班或甲方查阅时快速抓取关键信息。 |
| **报备内容可执行** | 「临时忽略XX专线相关的网络中断告警」写清了操作含义和目的（避免触发冗余应急），执行人知道具体怎么做、做到何时。 |
| **保障措施闭环** | 先写「施工完成后现场验证、确认恢复」，再写「若未恢复则切备线」，时间点（X:XX前）与兜底方案都有，便于收件人及甲方评估风险。 |

### 可补充（视单位要求选用）

- **落款与日期**：报备人、报备部门、日期，便于与割接记录、甲方报备材料对应留痕。
- **如需抄送**：若需同步甲方对接人、运维值班等，可注明抄送对象，确保割接窗口内各方信息一致。
- **恢复后闭环**：若制度要求「割接结束后再发一封恢复确认」，可在文末或制度中约定「恢复后将另行邮件/群内通报」，形成完整闭环。
