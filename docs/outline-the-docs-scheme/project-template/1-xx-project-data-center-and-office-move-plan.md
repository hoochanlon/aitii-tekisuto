风险与应急准备：详见机房搬迁应急预案（附件X）

##### (2）设备搬迁

第一步：所有设备配置、监控录像、服务器系统等进行备份，并验证验证备份数据的完整性和可用性。

第二步：将专线运营商设备移至2楼新机房，测试专线是否可用，不可用则回退待运营商解决问题。

第三步：将主备防火墙、主备核心交换机、服务器拆除下线搬迁至2楼机房。

第四步：将拆除设备按机柜内设备规划表依次上架，并按照搬迁后规划网络设备连接表将各个楼层的生产汇聚交换机接入核心并引入运营商专线，测试各个楼层职场生产电脑能否正常访问业务系统、软电话是否能正常登录、服务器系统及功能是否可用。

第五步：2楼监控接入交换机调整为监控核心，将1楼监控核心调整为监控接入，改造完成后将各个楼层监控接入交换机接入核心并测试，同时进行监控录像机及各个楼层摄像头工作状态检查，检查是否正常显示画面，录像能否正常储存。

##### （3）主备线切换验证

第一步：按顺序依次进行下线主防火墙、主核心、各楼层主汇聚，模拟在仅有备线及备用设备可用的情况下能否正常进行生产作业，如有故障及时修复。

第二部：每下线一台设备均测试各个楼层职场生产电脑能否正常访问业务系统、软电话是否能正常登录、服务器系统及功能确认备用设备能否够迅速、无缝地接管，并记录验证结果。

第二步：确认在仅有备线及备用设备可用的情况下可正常进行生产作业后，依次从下到上进行恢复，恢复后测试各个楼层职场生产电脑能否正常访问业务系统、软电话是否能正常登录、服务器系统及功能是否可用。

各项工作完成后，进行系统功能测试，并安排技术人员进行现场值守，保障业务生产正常。

##  第三章·机房搬迁应急预案

### 一、目的

本应急预案旨在针对机房搬迁过程中可能出现的风险，明确处置思路及流程，确保人员安全、数据完整、业务连续。

### 二、分类及应急处置流程

#### （1）网络

**新主机房内专线网络不通或不稳定**

处置原则：专线网络连通性和稳定性进行确认之后，机房设备才搬迁；若连通性和稳定性到搬迁次日十二点仍不能解决，搬迁延期进行。

处置流程：

1. 第一时间进行专线报修。
2. 要求运营商提供故障原因以及修复时间，同时上报领导。
3. 搬迁次日十二点前故障仍不能解决，搬迁延期。
4. 将专线网络进行复原，并进行主备线切换验证，验证正常后恢复使用。

#### （2）硬件

**网络设备损坏**

处置原则：使用办公网设备替换，并向总部紧急调取设备，待设备到位后更换。

处置流程：

1. 发现网络设备损坏后通知总部需调取设备。
2. 将损坏设备备份配置导入替换办公网设备，上架验证正常后投入使用。
3. 调取设备到位后将原有设备配置导入测试正常后进行替换。

**监控录像机损坏**

预防措施：关机确认硬盘停转后下架，并做防撞包装后搬迁。

处置原则：关机下线保存并通知厂商维修，并将2楼监控摄像头分流到另两台监控录像机。

处置流程：

1. 确认设备损坏后将2楼监控摄像头分流到另两台监控录像机。
2. 对摄像头画面是否正常进行全量确认。
3. 通知厂商维修损坏设备。
4. 损坏设备修复后上架验证，验证正常投入使用。

 ### 三、搬迁过程的合规及风险管理

1、运输安全保障：使用防震材料包装设备，确保设备在运输过程中的安全。每台设备均需贴上标签，标签内容包括设备名称、型号、清单编号和搬迁前使用位置等信息，保障后续安装和调试。使用手推车，将物品从20楼单箱依次通过货运直梯运输至22楼指定职场。
